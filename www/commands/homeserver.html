Homeserver · Neo

Fun fact: you are already talking to it. This portfolio is served from a ZimaBoard in my living room—not from some corporate cloud. Same box runs my photos, files, media, DNS blocking, and a pile of open-source tools I actually use every day.

The stack is Neo: a declarative NixOS homeserver I built so self-hosting is reproducible, revertable, and recoverable—without babysitting Docker Compose forever. Opinionated defaults, automatic HTTPS, double auth, auto-updates, plugins for everything that is too opinionated... Data stays on my machine. Port 443 in; everything else locked down.

Live on this machine (plugins)

Neo is modular: extra apps ship as plugins on the same secure front door. Two of mine are public right now—including the site you are on:

  • portrait.damo4mf20.ch — this portfolio (plugin: portrait)
  • rubiks.damo4mf20.ch — Rubik’s cube solver UI (plugin / project: rubiks) · also projects rubiks
  • Neo — the homeserver framework itself (open source, PRs welcome)
  • highsea.neo — media / *arr stack as a Neo plugin (Jellyfin, Sonarr, …)

Why bother?

Because renting forever is boring. Open source on hardware you own means you can inspect it, keep it, and move it. Neo is my attempt to make that accessible: same reproducibility story as Nix, packaged so you are not a full-time reverse-proxy engineer. Star it, fork it, break it, improve it.
